## 一、系统登录

1. 完成[本地运行](#本地运行)或在服务器部署后，打开浏览器访问对应地址
2. 在弹出的认证对话框中输入用户名和密码
3. 登录成功后进入主界面

## 二、界面概览

主界面分为五个功能标签页：

| 标签页 | 功能说明 |
|--------|----------|
| **概览** | 查看所有块设备、挂载点和磁盘空间使用情况 |
| **挂载** | 挂载新设备、卸载已有设备 |
| **文件系统** | 创建文件系统（格式化）、扩展文件系统空间 |
| **目录** | 浏览目录、创建/删除目录、调整权限 |
| **审计** | 查看所有管理操作的历史记录 |

顶部工具栏右侧有 **刷新按钮（↻）**，点击可刷新当前所有数据。

## 三、功能详解

### 3.1 概览 — 设备与空间

**用途**：一目了然地查看系统所有存储设备状态和磁盘使用情况。

**显示内容**：

**设备列表**（来自 `lsblk`）：
- **设备**：设备路径（如 `/dev/sda`、`/dev/nvme0n1`）
- **类型**：disk（磁盘）、part（分区）、lvm（逻辑卷）等
- **大小**：设备容量
- **文件系统**：已识别的文件系统类型（ext4、xfs、btrfs 等）
- **挂载点**：设备当前挂载位置

**空间使用**（来自 `df -hT`）：
- **文件系统**：文件系统设备名
- **类型**：文件系统类型
- **大小/已用/可用**：容量信息
- **使用率**：使用百分比（高使用率会标红提示）
- **挂载点**：挂载目录

> 💡 **提示**：子设备（如分区）会在父设备下方缩进显示，形成树状结构。

---

### 3.2 挂载 — 设备挂载与卸载

#### 3.2.1 挂载设备

**适用场景**：新硬盘分区、U 盘、网络存储等需要接入系统时。

**操作步骤**：

1. 切换到 **"挂载"** 标签页
2. 在 **"挂载设备"** 表单中填写：
   - **设备路径**：如 `/dev/sdb1`（必须位于 `/dev/` 下）
   - **挂载点**：如 `/mnt/data`（必须为绝对路径）
   - **文件系统**：可选，留空则自动识别
   - **挂载参数**：可选，如 `defaults,noatime,ro`
3. 点击 **"挂载"** 按钮
4. 操作成功后，右下角会弹出提示

**示例**：
```
设备路径: /dev/sdb1
挂载点: /mnt/data
文件系统: ext4
挂载参数: defaults
```

#### 3.2.2 卸载设备

**适用场景**：需要安全移除设备前，或调整挂载配置时。

**操作步骤**：

1. 在 **"卸载设备"** 表单中填写：
   - **设备或挂载点**：可以是设备路径（如 `/dev/sdb1`）或挂载点路径（如 `/mnt/data`）
   - **强制卸载**：勾选后使用 `-f` 参数强制卸载（设备忙时可用，但可能导致数据丢失）
2. 点击 **"卸载"** 按钮

> ⚠️ **警告**：卸载前请确保没有进程正在访问该挂载点，否则可能失败。

---

### 3.3 文件系统 — 格式化与扩容

#### 3.3.1 创建文件系统（格式化）

**适用场景**：新硬盘分区首次使用、需要清空并重新格式化设备时。

**操作步骤**：

1. 切换到 **"文件系统"** 标签页
2. 在 **"创建文件系统"** 表单中填写：
   - **设备路径**：如 `/dev/sdb1`
   - **类型**：选择 `ext4`、`xfs`、`btrfs`、`ext3` 或 `ext2`
   - **卷标**：可选，如 `DATA`
   - **覆盖已有签名**：勾选后强制格式化（会清除设备上的所有数据！）
3. 点击红色的 **"格式化"** 按钮

> ⚠️ **危险操作**：格式化会**永久删除**设备上的所有数据！请再三确认设备路径无误。

**支持的文件系统**：

| 类型 | 特点 | 适用场景 |
|------|------|----------|
| ext4 | Linux 默认，稳定成熟 | 通用存储 |
| xfs | 高性能，大文件友好 | 大数据、日志存储 |
| btrfs | 现代特性，快照支持 | 高级文件管理 |
| ext3/ext2 | 兼容旧系统 | 遗留系统兼容 |

#### 3.3.2 扩展空间（扩容）

**适用场景**：逻辑卷或分区扩容后，需要扩展文件系统以利用新增空间。

**操作步骤**：

1. 在 **"扩展空间"** 表单中填写：
   - **设备路径**：如 `/dev/sdb1`
   - **类型**：文件系统类型
   - **挂载点**：xfs 和 btrfs 扩容时需要提供挂载点（如 `/mnt/data`），ext 系列不需要
2. 点击 **"扩容"** 按钮

**不同类型注意事项**：

- **ext2/3/4**：直接对设备执行 `resize2fs`，无需挂载点
- **xfs**：必须在已挂载状态下执行，需要提供挂载点
- **btrfs**：必须在已挂载状态下执行，需要提供挂载点

> 💡 **前置条件**：扩容文件系统前，底层分区或逻辑卷必须已经扩展（通过 `fdisk`、`lvextend` 等工具）。本系统只负责扩展文件系统本身。

---

### 3.4 目录 — 浏览与管理

#### 3.4.1 浏览目录

**操作步骤**：

1. 切换到 **"目录"** 标签页
2. 在顶部 **"当前目录"** 输入框中输入路径（如 `/mnt`）
3. 点击 **"打开"** 按钮
4. 下方表格显示该目录下的所有文件和子目录：
   - **名称**：文件/目录名
   - **类型**：目录 或 文件
   - **权限**：八进制权限（如 `0o755`）
   - **属主**：`用户名:用户组`
   - **大小**：字节数

> 💡 **提示**：目录会排在前面，文件排在后面，均按字母顺序排列。

#### 3.4.2 创建目录

**操作步骤**：

1. 在 **"创建目录"** 表单中填写：
   - **目录路径**：如 `/mnt/data/logs`（必须为绝对路径）
2. 点击 **"创建"** 按钮
3. 系统会自动创建父目录（等同于 `mkdir -p`）

> ⚠️ **安全限制**：无法操作系统关键目录（如 `/`、`/bin`、`/etc`、`/usr`、`/var` 等）。

#### 3.4.3 删除目录

**操作步骤**：

1. 在 **"删除目录"** 表单中填写：
   - **目录路径**：如 `/mnt/data/logs`
   - **递归删除**：勾选后删除目录及其所有内容（等同于 `rm -r`）
2. 点击红色的 **"删除"** 按钮

> ⚠️ **危险操作**：删除操作**不可恢复**！递归删除会清除目录下所有文件和子目录。

#### 3.4.4 权限调整

**操作步骤**：

1. 在 **"权限调整"** 表单中填写：
   - **路径**：要调整权限的文件或目录
   - **权限**：八进制模式，如 `755`、`0644`
   - **用户**：可选，新属主用户名
   - **用户组**：可选，新属组名
2. 点击 **"应用"** 按钮

**常用权限参考**：

| 权限 | 说明 |
|------|------|
| `755` | 所有者可读写执行，其他用户可读执行（目录默认） |
| `644` | 所有者可读写，其他用户只读（文件默认） |
| `700` | 仅所有者可访问 |
| `777` | 所有人可读写执行（慎用） |

> 💡 **提示**：输入的用户名和用户组必须是系统中已存在的（系统会验证）。

---

### 3.5 审计 — 操作日志

**用途**：追踪所有管理操作，便于故障排查和安全审计。

**显示内容**：

| 字段 | 说明 |
|------|------|
| **时间** | 操作执行时间 |
| **动作** | 操作类型（mount、unmount、make_filesystem 等） |
| **命令** | 实际执行的系统命令 |
| **结果** | 成功 或 失败及返回码 |

**特性**：
- 按时间倒序排列（最新的在最上面）
- 默认显示最近 100 条记录
- 所有写操作（挂载、卸载、格式化、扩容、目录创建/删除、权限修改）均自动记录
- 命令输出（stdout/stderr）也会被截断保存（最多 4000 字符）

> 🔒 **安全提示**：审计日志存储在 SQLite 数据库中，建议定期备份 `/opt/storage-manager/data/storage_manager.sqlite3`。

## 四、操作示例流程

### 示例 1：挂载一块新硬盘

```
1. 插入/连接新硬盘
2. 在"概览"页确认设备出现（如 /dev/sdb1）
3. 切换到"挂载"页
4. 填写：设备路径=/dev/sdb1，挂载点=/mnt/data，文件系统=ext4
5. 点击"挂载"
6. 在"概览"页确认挂载成功
7. 切换到"目录"页，浏览 /mnt/data 确认可访问
```

### 示例 2：格式化并挂载 U 盘

```
1. 插入 U 盘
2. 在"概览"页确认设备（如 /dev/sdc1）
3. 切换到"文件系统"页
4. 填写：设备路径=/dev/sdc1，类型=ext4，勾选"覆盖已有签名"
5. 点击"格式化"（⚠️ 数据将被清空）
6. 切换到"挂载"页
7. 填写：设备路径=/dev/sdc1，挂载点=/mnt/usb
8. 点击"挂载"
```

### 示例 3：为 Web 服务创建日志目录

```
1. 切换到"目录"页
2. 在"创建目录"中填写：/var/www/logs
3. 点击"创建"
4. 在"权限调整"中填写：
   - 路径=/var/www/logs
   - 权限=755
   - 用户=www-data
   - 用户组=www-data
5. 点击"应用"
6. 在"审计"页查看操作记录
```

## 五、安全注意事项

### 5.1 认证安全

- 生产环境务必修改默认密码
- 不要在未加密的 HTTP 网络下直接暴露服务
- 当前通过 Nginx HTTPS 访问，后端 8010 端口仅监听本地

### 5.2 操作安全

| 风险等级 | 操作 | 建议 |
|----------|------|------|
| 🔴 高危 | 格式化设备 | 操作前确认设备路径，避免误格式化系统盘 |
| 🔴 高危 | 递归删除目录 | 确认路径正确，关键系统目录已被保护 |
| 🟡 中危 | 强制卸载 | 可能导致数据损坏，仅在其他方式失败时使用 |
| 🟢 低危 | 查看操作 | 只读操作，无风险 |

### 5.3 关键目录保护

以下目录**禁止**进行创建/删除/权限修改操作：

```
/ /bin /boot /dev /etc /lib /lib64
/proc /root /run /sbin /sys /usr /var
```

## 六、故障排查

### 问题 1：挂载失败 "设备忙"

**原因**：有进程正在使用该挂载点。  
**解决**：
```bash
# 在服务器终端查看占用进程
lsof /mnt/data
fuser -mv /mnt/data
```

### 问题 2：格式化失败 "设备已挂载"

**原因**：设备当前已挂载，无法直接格式化。  
**解决**：先卸载设备，再格式化。

### 问题 3：权限修改失败 "用户不存在"

**原因**：输入的用户名或用户组不存在于系统。  
**解决**：先创建用户/用户组，或在"权限调整"中留空不修改属主。

### 问题 4：xfs 扩容失败

**原因**：xfs 文件系统必须在已挂载状态下扩容。  
**解决**：确保设备已挂载，并在表单中填写挂载点。

### 问题 5：服务无法访问

**排查步骤**：
```bash
# 在服务器上执行
systemctl status storage-manager    # 查看服务状态
journalctl -u storage-manager -n 50 # 查看日志
ss -tlnp | grep 8010                # 确认端口监听
nginx -t                            # 检查 Nginx 配置
```

## 七、技术架构

```
用户浏览器 → Nginx (443 HTTPS)
                ↓
         /storage-manager/ 反向代理
                ↓
         uvicorn (127.0.0.1:8010)
                ↓
         FastAPI (app.main:app)
                ↓
    ┌──────────┴──────────┐
    ↓                     ↓
storage.py           SQLAlchemy ORM
(系统命令)            ↓
    ↓              SQLite
lsblk/df/           (审计日志)
mount/umount/
mkfs/resize2fs
```
