三、功能详解

3.2 挂载 — 设备挂载与卸载

3.2.1 挂载设备

适用场景：新硬盘分区、U 盘、网络存储等需要接入系统时。

  1. 切换到 "挂载" 标签页
  2. 在 "挂载设备" 表单中填写：
    • 设备路径：如 /dev/sdb1（必须位于 /dev/ 下）
    • 挂载点：如 /mnt/data（必须为绝对路径）
    • 文件系统：可选，留空则自动识别
    • 挂载参数：可选，如 defaults,noatime,ro
  3. 点击 "挂载" 按钮
  4. 操作成功后，右下角会弹出提示

3.2.2 卸载设备

适用场景：需要安全移除设备前，或调整挂载配置时。

  1. 在 "卸载设备" 表单中填写：
    • 设备或挂载点：可以是设备路径或挂载点路径
    • 强制卸载：勾选后使用 -f 参数强制卸载
  2. 点击 "卸载" 按钮
⚠️ 警告：卸载前请确保没有进程正在访问该挂载点，否则可能失败。

3.3 文件系统 — 格式化与扩容

3.3.1 创建文件系统（格式化）

适用场景：新硬盘分区首次使用、需要清空并重新格式化设备时。

  1. 切换到 "文件系统" 标签页
  2. 在 "创建文件系统" 表单中填写：
    • 设备路径：如 /dev/sdb1
    • 类型：选择 ext4、xfs、btrfs、ext3 或 ext2
    • 卷标：可选，如 DATA
    • 覆盖已有签名：勾选后强制格式化（会清除所有数据！）
  3. 点击红色的 "格式化" 按钮
🔴 危险操作：格式化会永久删除设备上的所有数据！请再三确认设备路径无误。
类型特点适用场景
ext4Linux 默认，稳定成熟通用存储
xfs高性能，大文件友好大数据、日志存储
btrfs现代特性，快照支持高级文件管理
ext3/ext2兼容旧系统遗留系统兼容

3.3.2 扩展空间（扩容）

适用场景：逻辑卷或分区扩容后，需要扩展文件系统以利用新增空间。

  1. 在 "扩展空间" 表单中填写：
    • 设备路径：如 /dev/sdb1
    • 类型：文件系统类型
    • 挂载点：xfs 和 btrfs 扩容时需要提供挂载点
  2. 点击 "扩容" 按钮
💡 前置条件：扩容文件系统前，底层分区或逻辑卷必须已经扩展。本系统只负责扩展文件系统本身。

3.4 目录 — 浏览与管理

3.4.1 浏览目录

  1. 切换到 "目录" 标签页
  2. 在顶部 "当前目录" 输入框中输入路径（如 /mnt）
  3. 点击 "打开" 按钮
  4. 下方表格显示该目录下的所有文件和子目录

3.4.2 创建目录

  1. 在 "创建目录" 表单中填写目录路径（如 /mnt/data/logs）
  2. 点击 "创建" 按钮
  3. 系统会自动创建父目录（等同于 mkdir -p）

3.4.3 删除目录

  1. 在 "删除目录" 表单中填写目录路径
  2. 勾选 "递归删除" 以删除目录及其所有内容
  3. 点击红色的 "删除" 按钮
🔴 危险操作：删除操作不可恢复！递归删除会清除目录下所有文件和子目录。

3.4.4 权限调整

  1. 在 "权限调整" 表单中填写：
    • 路径：要调整权限的文件或目录
    • 权限：八进制模式，如 755、0644
    • 用户/用户组：可选，新属主和属组
  2. 点击 "应用" 按钮
权限说明
755所有者可读写执行，其他用户可读执行（目录默认）
644所有者可读写，其他用户只读（文件默认）
700仅所有者可访问
777所有人可读写执行（慎用）

3.5 审计 — 操作日志

用途：追踪所有管理操作，便于故障排查和安全审计。

字段说明
时间操作执行时间
动作操作类型（mount、unmount、make_filesystem 等）
命令实际执行的系统命令
结果成功 或 失败及返回码

四、操作示例

示例 1：挂载一块新硬盘

1. 插入/连接新硬盘
2. 在"概览"页确认设备出现（如 /dev/sdb1）
3. 切换到"挂载"页
4. 填写：设备路径=/dev/sdb1，挂载点=/mnt/data，文件系统=ext4
5. 点击"挂载"
6. 在"概览"页确认挂载成功
7. 切换到"目录"页，浏览 /mnt/data 确认可访问

示例 2：格式化并挂载 U 盘

1. 插入 U 盘
2. 在"概览"页确认设备（如 /dev/sdc1）
3. 切换到"文件系统"页
4. 填写：设备路径=/dev/sdc1，类型=ext4，勾选"覆盖已有签名"
5. 点击"格式化"（⚠️ 数据将被清空）
6. 切换到"挂载"页
7. 填写：设备路径=/dev/sdc1，挂载点=/mnt/usb
8. 点击"挂载"

示例 3：为 Web 服务创建日志目录

1. 切换到"目录"页
2. 在"创建目录"中填写：/var/www/logs
3. 点击"创建"
4. 在"权限调整"中填写：路径=/var/www/logs，权限=755，用户=www-data，用户组=www-data
5. 点击"应用"
6. 在"审计"页查看操作记录

五、安全注意事项

5.1 认证安全

  • 生产环境务必修改默认密码
  • 当前通过 Nginx HTTPS 访问，后端 8010 端口仅监听本地

5.2 操作安全

风险等级操作建议
高危格式化设备操作前确认设备路径，避免误格式化系统盘
高危递归删除目录确认路径正确，关键系统目录已被保护
中危强制卸载可能导致数据损坏，仅在其他方式失败时使用
低危查看操作只读操作，无风险

5.3 关键目录保护

以下目录禁止进行创建/删除/权限修改操作：

/ /bin /boot /dev /etc /lib /lib64
/proc /root /run /sbin /sys /usr /var

六、故障排查

问题原因解决
挂载失败 "设备忙"有进程正在使用该挂载点lsof /mnt/data 或 fuser -mv /mnt/data
格式化失败 "设备已挂载"设备当前已挂载先卸载设备，再格式化
权限修改失败 "用户不存在"用户名/用户组不存在先创建用户/用户组，或留空不修改
xfs 扩容失败xfs 必须在已挂载状态下扩容确保设备已挂载，并填写挂载点

服务无法访问排查

systemctl status storage-manager    # 查看服务状态
journalctl -u storage-manager -n 50 # 查看日志
ss -tlnp | grep 8010                # 确认端口监听
nginx -t                            # 检查 Nginx 配置
